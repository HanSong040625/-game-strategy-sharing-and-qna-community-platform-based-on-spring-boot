## 5. API调用方式

### 5.1 前端调用示例

**用户登录**：
```javascript
this.$axios.post('/api/auth/login', {
  username: 'test',
  password: '123456'
}).then(response => {
  if (response.data.code === 200) {
    // 登录成功
    localStorage.setItem('username', response.data.data.username);
    localStorage.setItem('isAdmin', response.data.data.isAdmin);
  }
});
```

**获取游戏列表**：
```javascript
this.$axios.get('/api/game/list')
  .then(response => {
    this.games = response.data;
  });
```

**管理员获取用户列表（带分页）**：
```javascript
this.$axios.get('/api/admin/users', {
  params: {
    page: this.currentPage,
    pageSize: this.pageSize
  }
}).then(response => {
  if (response.data.code === 200) {
    this.users = response.data.data;
    this.total = response.data.total;
  }
});
```

### 5.2 后端服务调用

**Service层调用**：
```java
// 在Controller中调用Service方法
@Autowired
private UserService userService;

@PostMapping("/register")
public ResponseEntity<String> register(@RequestParam String username, 
                                      @RequestParam String password, 
                                      @RequestParam String email) {
    try {
        User user = userService.register(username, password, email);
        return ResponseEntity.ok("{\"code\":200,\"message\":\"注册成功\",\"data\":null}");
    } catch (Exception e) {
        return ResponseEntity.status(HttpStatus.BAD_REQUEST)
                .body("{\"code\":400,\"message\":\"" + e.getMessage() + "\",\"data\":null}");
    }
}
```

## 6. 业务流程

### 6.1 用户注册流程

1. 前端提交注册信息（用户名、密码、邮箱）
2. 后端接收请求，调用 `UserService.register()` 方法
3. 验证参数合法性
4. 检查用户名和邮箱是否已存在
5. 对密码进行加密
6. 创建用户并保存到数据库
7. 返回注册结果

### 6.2 游戏管理流程

1. 管理员登录系统
2. 访问游戏管理页面
3. 可以执行以下操作：
   - 添加新游戏：填写游戏信息，上传图片
   - 修改游戏：更新游戏信息
   - 删除游戏：删除指定游戏
   - 设置推荐：设置游戏为推荐状态
4. 所有操作都会记录操作日志

### 6.3 私信发送流程

1. 登录用户选择接收者
2. 输入消息内容并发送
3. 后端接收请求，调用 `PrivateMessageService.sendMessage()` 方法
4. 创建私信记录
5. 通过WebSocket发送实时通知给接收者
6. 返回发送结果

## 8. 常见问题

### 8.1 登录失败

**可能原因**：
- 用户名或密码错误
- 用户账号被封禁
- 数据库连接异常

**解决方案**：
- 检查用户名和密码是否正确
- 联系管理员检查账号状态
- 检查数据库连接

### 8.2 上传图片失败

**可能原因**：
- 文件大小超过限制（2MB）
- 文件格式不支持（仅支持JPG、JPEG、PNG、GIF）
- 服务器存储空间不足

**解决方案**：
- 压缩图片大小
- 转换为支持的格式
- 检查服务器存储空间

### 8.3 分页功能不工作

**可能原因**：
- 前端未传递分页参数
- 后端未实现分页逻辑
- 数据库查询异常

**解决方案**：
- 确保前端传递 `page` 和 `pageSize` 参数
- 检查后端分页实现
- 检查数据库连接和查询语句
